Discourse coherence

Language does not normally consist of isolated, unrelated sentences, but instead of collocated, structured, coherent groups of sentences. One possibility offered by computationnal linguistic is to compute the coherence of a discourse.

The coherence take the form of a value between 1 and 0, between two sentences, and can ne computed using the package LSAfun. For example :

data(wonderland)

coherence ("There was certainly too much of it in the air. Even the Duchess
sneezed occasionally; and as for the baby, it was sneezing and howling
alternately without a moment's pause. The only things in the kitchen
that did not sneeze, were the cook, and a large cat which was sitting on
the hearth and grinning from ear to ear.",
tvectors=wonderland)
## $local
## [1] 0.5666128 0.5948014
## 
## $global
## [1] 0.5807071
# }

Considering that sentences are related between them inside section, the analysis of the coherence of a scientific article could highlight some breakpoints, corresponding to differents section.

Coherence of sentences :

The article used is the first one in x, namely Adam et al, 2010, in the Biodistribution folder. The lines below isolate the sentences of the first documents, and remove the one with less than 8 caracters, to make the output of the analysis more clear.

load("EN_100k_lsa.rda")
lsa=EN_100k_lsa

x <- readRDS(file = "annotation_gum.rds")
x <- as.data.frame(x)


sentences<-x[which(x$doc_id=="doc1"),]$sentence
sentences<-unique(sentences)
sentences<-sentences[which(nchar(sentences)>8)]
length(sentences)
## [1] 512
for (sentence in sentences)
{
  sentence<-paste(sentence, "S3P4RAT0R", sep = "", collapse = NULL)
}
#"S3P4RAT0R" is a way to avoid the default behavior of the function which trim the input with the dot.
#otherwise, the coherence function output an object with a different lemght than the number of sentences.

Adding “S3P4RAT0R” is a way to avoid the default behavior of the function which trim the input with the dot. Otherwise, the coherence function output an object with a different lemght than the number of sentences.

Coherence of paragraph

An other layer of complexity in the datastructure for NLP provided by UDpipe is the paragraph. The following section explore the same approach but for paragraph. Extracting them from the text require a bit of pre-processing.

x<-x[which(x$doc_id=="doc1"),]

paragraph_id<-x[which(x$doc_id=="doc1"),]$paragraph_id
paragraph_id<-unique(paragraph_id)
max(paragraph_id)
## [1] 620
all_paragraph<-c()

for (id in paragraph_id)
{
  paragraph<-x[which(x$paragraph_id==id),]$sentence
  paragraph<-unique(paragraph)
  paragraph<-do.call(paste, c(as.list(paragraph), sep = ""))
  all_paragraph<-c(all_paragraph, paragraph)
}


all_paragraph<-all_paragraph[which(nchar(all_paragraph)>8)]
txt_article<-do.call(paste, c(as.list(all_paragraph), sep = "S3P4RAT0R"))


coherence<-coherence(txt_article, split=c("S3P4RAT0R"), tvectors=lsa,breakdown=FALSE)

## no element of s found in rownames(tvectors) for some sentences s in x
